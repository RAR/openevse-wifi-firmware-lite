# Slice 1.5 — Control seam: claim/setpoint surface (design)

**Status:** design for review.
**Worktree:** `/home/rar/oevse/openevse-juicebox-lite` (branch `feature/juicebox-lite`).
**Context:** second slice of the JuiceBox-lite reuse roadmap. Slice 1 added a single
`setChargeCurrent` setpoint; this slice grows `LiteEvseBackend` into the **claim/setpoint control
surface** the standard-fw control cluster (`manual`/`limit`/`temp_throttle`/`divert`/`current_shaper`/
`scheduler`/`mqtt`) drives `EvseManager` through — so those modules become **gated lifts** in their
own later slices instead of rewrites. See the lift audit
`docs/superpowers/notes/2026-06-13-standard-fw-lift-audit.md` and the protocol RE
`docs/superpowers/notes/2026-06-13-juicebox-protocol-re.md`.

## Goal
Introduce a minimal, faithful, `EvseManager`-compatible control manager (`LiteEvseManager`) over
`LiteEvseBackend` that provides claim registration + per-field priority arbitration + the getter/event
surface the control cluster expects, maps the arbitrated result onto the JuiceBox `$SL` setpoint, and
subsumes Slice 1's configured charge limit as the manager's base target. Prove it end-to-end by
lifting one canary module (`manual`). Defer the heavier module lifts to their own slices.

## Hardware reality (from the protocol RE — drives the `Disabled` mapping)
The JuiceBox ATmega has **no inbound host start/stop command** (full inbound dispatch decoded; no
`$SE`/`$SP`/`$SC`/sleep/standby type). The host's only charge lever is the amps setpoint (`$SL`-family,
value 0–80, MCU floors `<6` to 6 A). The comm watchdog does **not** stop charging (offline falls back
to the last setpoint). Therefore the seam's `Disabled` state cannot truly open the contactor today;
it maps to **minimum amps (6 A)**, isolated in a single backend method (`setState`) so the real stop
command can be wired in later once captured from a fresh unit via live UART (deferred by decision
2026-06-13). (RE correction logged: the state helper `0x3b72(arg==0)`→state 3, `arg!=0`→state 2.)

## Architecture
```
web_server_lite / future modules (manual,limit,divert,...)
        │  claim()/release()/getState()/getChargeCurrent()/setTarget*()
        ▼
   LiteEvseManager   ── owns: _target (EvseProperties) + claim registry[8] + LiteEvseBackend&
        │  on any change: resolved = arbitrate(_target, claims);
        │                 backend.setState(resolved.state);
        │                 backend.setChargeCurrent(clamp(min(resolved.cc, resolved.mc)))
        ▼
   LiteEvseBackend (JuiceBoxBackend)
        │  setChargeCurrent → $SL002:NN ;  setState(Disabled) → min-amps (TODO real stop)
        ▼
   ATmega ($-protocol)
```
A compatibility alias `using EvseManager = LiteEvseManager;` (in a lite header) lets future module
lifts compile against `EvseManager *` unmodified.

## Components & changes

### 1. Claim value types (new, mirror upstream names/types)
- **Files:** `src/lite/lite_evse_claims.h` (new).
- `enum class EvseState : uint8_t { None, Active, Disabled };` — the claim *target* state. Distinct
  from the existing device-facing `LiteEvseState` (which reports the JuiceBox `$ES` S/fault).
- `EvseClient` as `uint32_t` with the verbatim upstream values: `EvseClient_OpenEVSE_Manual`
  (0x00010001), `_Divert` (0x00010002), `_Boost` (0x00010003), `_Schedule` (0x00010004), `_Limit`
  (0x00010006), `_Error` (0x00010007), `_Ohm` (0x00010008), `_OCPP` (0x00010009), `_RFID` (0x0001000A),
  `_MQTT` (0x0001000B), `_Shaper` (0x0001000C), `_TempThrottle` (0x0001000D); `EvseClient_NULL`
  (UINT32_MAX).
- Priority constants (verbatim): `EvseManager_Priority_Default`=10, `_Divert`=50, `_Timer`=100,
  `_Boost`=200, `_API`=500, `_MQTT`=500, `_Ohm`=500, `_Manual`=1000, `_RFID`=1030, `_OCPP`=1050,
  `_Limit`=1100, `_Safety`=5000, `_Error`=10000.

### 2. EvseProperties (new, mirror upstream fields)
- **Files:** `src/lite/lite_evse_properties.{h,cpp}` (new); native test `test/test_lite_evse_props/`.
```cpp
class EvseProperties {
public:
  EvseProperties();                      // state=None, charge_current=UINT32_MAX (unset),
                                         // max_current=UINT32_MAX (unset), auto_release=false
  explicit EvseProperties(EvseState state);
  void clear();
  EvseState getState() const;            void setState(EvseState s);
  uint32_t getChargeCurrent() const;     void setChargeCurrent(uint32_t a);   // UINT32_MAX = unset
  uint32_t getMaxCurrent() const;        void setMaxCurrent(uint32_t a);      // UINT32_MAX = unset
  bool isAutoRelease() const;            void setAutoRelease(bool b);
  bool hasChargeCurrent() const;         // charge_current != UINT32_MAX
  bool hasMaxCurrent() const;            // max_current   != UINT32_MAX
  EvseProperties &operator=(const EvseProperties &rhs);
  void deserialize(JsonObject &obj);     void serialize(JsonObject &obj) const;  // mirror upstream keys
};
```
- Field semantics match upstream `evse_man.h`: `charge_current`/`max_current` of `UINT32_MAX` mean
  "unset / no opinion" (so a claim can affect only `state`, only `current`, etc.).

### 3. Arbitration (pure, native-tested — the testable core)
- **Files:** `src/lite/lite_evse_arbitrate.{h,cpp}` (new); native test `test/test_lite_evse_arbitrate/`.
```cpp
struct EvseClaim { EvseClient client; int priority; EvseProperties props; bool active; };
// Per-field highest-priority-wins, starting from `target`. Only claims with active==true and a
// field "set" (hasChargeCurrent / hasMaxCurrent / state!=None) override that field; among setters,
// the highest priority wins. Returns the resolved properties (state always set: target.state if no
// claim sets state).
EvseProperties lite_evse_arbitrate(const EvseProperties &target,
                                   const EvseClaim *claims, size_t n_claims);
```
- State precedence: among claims that set `state` (state != None), highest priority wins; ties → last
  wins (deterministic by array order, documented). `Disabled` does not auto-win — a higher-priority
  `Active` claim beats a lower-priority `Disabled`, matching upstream per-field semantics.
- Pure, no Arduino deps → compiled + unit-tested in the native doctest env (add to `[env:native]`).

### 4. LiteEvseManager (new)
- **Files:** `src/lite/lite_evse_manager.{h,cpp}` (new), `src/lite/lite_evse_compat.h` (the
  `using EvseManager = LiteEvseManager;` alias for future lifts).
- Owns: `EvseProperties _target;` `EvseClaim _claims[8];` `uint8_t _claimsVersion;`
  `LiteEvseBackend &_backend;` and the MicroTasks event listener lists.
- **Claim API (EvseManager-compatible):**
```cpp
bool claim(EvseClient client, int priority, EvseProperties &target); // false if registry full
bool release(EvseClient client);
bool clientHasClaim(EvseClient client) const;
uint8_t getClaimsVersion() const;
EvseProperties &getClaimProperties(EvseClient client); // client==NULL ⇒ _target; else the claim's props
```
- **Target API (Slice-1 integration):** `setTargetState(EvseState)`, `setTargetChargeCurrent(uint32_t)`,
  `setTargetMaxCurrent(uint32_t)` — `web_server_lite` updates the target through these instead of
  calling `backend.setChargeCurrent` directly.
- **Claim-aware getters:** `EvseState getState(EvseClient = EvseClient_NULL)`,
  `uint32_t getChargeCurrent(EvseClient = EvseClient_NULL)`,
  `uint32_t getMaxCurrent(EvseClient = EvseClient_NULL)` — `NULL` returns the *resolved* value; a
  client returns that claim's stored value.
- **Apply path:** any claim/release/target change calls a private `apply()`:
  `resolved = lite_evse_arbitrate(_target, _claims, 8);` then
  `_backend.setState(resolved.getState());`
  `_backend.setChargeCurrent(lite_clamp_charge_current(min(resolved cc, resolved mc), hard));`
  (reusing Slice-1's `lite_charge_policy`); bump `_claimsVersion`; fire `onStateChange` if the
  resolved state or device state changed.
- **Delegating getters** (pass through to backend): `isCharging()`, `getAmps()`, `getTemperature(i)`,
  `isTemperatureValid(i)`, `getMinCurrent()`, `getMaxHardwareCurrent()`, `getEvseState()`,
  `getVoltage()` (tier-2 stub).
- **Events:** `onStateChange(MicroTasks::EventListener*)`, `onSessionComplete(MicroTasks::EventListener*)`
  (MicroTasks already linked in the lite build). Fire `onStateChange` on resolved/device state change;
  `onSessionComplete` on a charging→not-charging device transition.

### 5. LiteEvseBackend additions
- **Files:** `src/lite/lite_evse_backend.h`, `src/lite/juicebox_backend.{h,cpp}`.
- Add to the interface + JuiceBox impl:
```cpp
virtual void setState(EvseState state) = 0;     // Active ⇒ drive setpoint; Disabled ⇒ stop
virtual bool isCharging() const = 0;            // from $ES S (charging states)
virtual int  getMinCurrent() const = 0;         // J1772 floor (6)
virtual int  getMaxHardwareCurrent() const = 0; // install rating (service-max)
virtual int  getTemperature() const = 0;        // $ES T (single sensor)
virtual bool isTemperatureValid() const = 0;    // $ES T present/in-range
virtual int  getEvseState() const = 0;          // raw $ES S
```
- `JuiceBoxBackend::setState`:
```cpp
void setState(EvseState s) override {
  // TODO(fresh-unit capture): the JuiceBox $-protocol has NO host stop command (RE-confirmed);
  // Disabled currently maps to the 6 A floor (cannot truly open the contactor). When the real
  // stop command is captured from a fresh unit, wire it here — this is the only place to change.
  _enabled = (s != EvseState::Disabled);
  // keepalive advertises _chargeLimit when enabled, else JB_MIN_CURRENT.
}
```
  `sendKeepalive()` sends `_enabled ? _chargeLimit : JB_MIN_CURRENT`. Keep `getChargeCurrent()`
  returning the *requested* `_chargeLimit` (so the manager's resolved value round-trips), and `$ES` A
  remains the MCU-reported amps via `getAmps()`.
- Tier-1 getters map to existing `$ES` fields (`isCharging`/`getEvseState` from S, `getTemperature`
  from T). `getMinCurrent`=6, `getMaxHardwareCurrent` = the Slice-1 `max_current_hard`. Tier-2
  (`getVoltage`, session energy/elapsed, vehicle) are stubbed (`getVoltage()` returns 0 / a const)
  with TODOs, filled when the modules that need them land.

### 6. Canary lift: manual
- **Files:** copy `src/manual.cpp`/`manual.h` from the standard tree into the lite build (gated
  `OPENEVSE_LITE`); wire via the `EvseManager` alias. Expose a tiny control: `GET/POST /override`
  (claim `EvseClient_OpenEVSE_Manual` at `EvseManager_Priority_Manual` with the requested state/current;
  release on clear) — mirrors upstream `web_server` manual override, minimal form.
- This proves the surface end-to-end: a manual claim overrides the base target and the resolved value
  reaches `$SL`.

### 7. Wiring
- **Files:** `src/lite/main_lite.cpp`, `src/lite/web_server_lite.cpp`.
- `main_lite`: construct `LiteEvseManager mgr(s_backend);` after `s_backend.begin()`; seed
  `mgr` target from Slice-1 config (replaces the direct `backend.setChargeCurrent` boot apply);
  pass `mgr` to `web_server_lite_begin`.
- `web_server_lite`: `/config` set path calls `mgr.setTargetChargeCurrent/​MaxCurrent` (and the boot
  apply seeds the target); `/status` reports resolved `getChargeCurrent()`/`getState()` plus the
  active-claim count for visibility.

## Data flow (examples)
- `claim(Divert, 50, {charge_current:12})` → arbitrate(target{cc:32,Active}, …) → resolved cc=12 →
  `setChargeCurrent(12)` → `$SL002:12`.
- `claim(Limit, 1100, {state:Disabled})` → resolved state=Disabled → `setState(Disabled)` → keepalive
  drops to 6 A (TODO real stop).
- `release(Limit)` → no state-setter left → resolved state=target.state(Active) → resumes target cc.
- `web POST /config?max_current_soft=20` → `setTargetChargeCurrent(20)` → if no higher claim, resolved
  cc=20 → `$SL002:20`.

## Error handling
- Registry full (8 active claims) → `claim()` returns `false` (logged in debug build); existing claims
  unaffected.
- No claims → resolved = `_target`.
- `charge_current > max_current` → resolved clamped to `max_current`, then to `[JB_MIN_CURRENT,
  JB_ABS_MAX]` via `lite_charge_policy`.
- `Disabled` → 6 A min (documented hardware limitation, pending fresh-unit capture).
- `release()` of an absent client → no-op, returns `false`.

## Testing
- **Native doctest (pure):**
  - `test/test_lite_evse_arbitrate/`: empty→target; single claim sets cc; higher-priority claim wins a
    field; `Disabled@1100` beats `Active@10` but loses to `Active@5000`; per-field independence (one
    claim sets state, another sets current); `max_current` clamps `charge_current`; full-registry (8);
    unset fields (`UINT32_MAX`) don't override.
  - `test/test_lite_evse_props/`: getters/setters, has-flags, `UINT32_MAX` unset semantics,
    serialize/deserialize round-trip.
- **On-device (JuiceBox 10.75.1.216):**
  1. Base: `/config?max_current_soft=24` → `$SL002:24`.
  2. Manual claim via `/override` (current=12) → `$SL002:12` (claim overrides base); `/status` shows
     the manual claim active.
  3. Manual claim state=Disabled → keepalive drops to `$SL002:06`; release → returns to base.
  4. Priority: a simulated higher-priority claim beats manual (exercised via a temporary test hook or
     the next module's lift).

## Follow-on
Each control module becomes a gated lift in its slice: it includes the `EvseManager` compat alias,
calls the same claim/getter API, and "just works" against `LiteEvseManager`. The JuiceBox `Disabled`
mapping is the one place to upgrade when the real stop command is captured.
